### Low

- **`apps/web/src/perf.test.ts:88-101` — `snapshot` memory-shape assertion is weaker than the comment implies.**
  `expect(snap.memory).toHaveProperty("usedJSHeapSize")` passes even when the value is `undefined`, because `toHaveProperty` only checks key presence by default. The follow-up `value === null || typeof === "number"` check then catches that, but combining the two means a regression where the shim returns `undefined` would only be flagged by the second assertion, not the first. Concrete fix: either drop the redundant `toHaveProperty` lines or strengthen them with the explicit value form, e.g. `expect(snap.memory.usedJSHeapSize).toSatisfy((v) => v === null || typeof v === "number")` (or just the existing union assertion alone is enough — the `toHaveProperty` line is decorative).
