### Low

- **`apps/web/src/panels/cursorOverlay.ts:48` — test-only reset hook is a
  regular named export.** `__resetCursorStrokeColorCache()` exists purely so
  unit tests can reset the new module-level cache, but it's exported on the
  same surface as `cursorStrokeColor` / `cursorXPx`, so production code can
  call it. The `__` prefix is a convention, not a guarantee. Fix: either
  guard with `import.meta.vitest` and put the hook behind that block, or move
  the cache + helper into a `__test__` sibling module that the production
  module re-imports only in test runs. At minimum, drop a `// @internal —
  tests only` JSDoc tag so the next reader doesn't grow a real call site.

- **`apps/web/src/panels/PlotPanel.tsx:106` — `axisStyle()` caches without a
  test reset hook, unlike `cursorStrokeColor`.** Both helpers were converted
  to module-level caches in the same commit, but only `cursorOverlay` got a
  reset escape hatch (`__resetCursorStrokeColorCache`). If any future test
  needs to assert PlotPanel re-reads tokens after a `--color-fg-2` change
  (e.g. when a real theme switch is added in v2), the cache will silently
  pin the old value and the test will pass spuriously. Fix: export
  `__resetAxisStyleCache()` for symmetry, or accept the asymmetry and add a
  `// no theme switch at v1` comment that points readers at the
  cursorOverlay precedent.

### Nits

- **`apps/web/src/panels/EnumPanel.tsx:97`, `MapPanel.tsx:122`,
  `PlotPanel.tsx:166`, `TablePanel.tsx:80` — `sources.length` in the deps
  array is fine, but the gate's behaviour on "user removes last source" is
  worth a one-line comment.** Because the cull is skipped when
  `sources.length === 0`, dragging the last source out keeps the binding
  alive; the binding is then either revived (channel re-appears) or culled
  (channel doesn't) on the next drop. That's the right product call, but
  it's not what a reader expects from "drop binding when channel
  disappears." Add a sentence to the existing comment block making the
  policy explicit.

- **`apps/web/src/state/store.ts:257` — `mintId(prefix)` ignores `prefix` in
  the `randomUUID` branch.** Production gets `550e8400-e29b-…`; jsdom tests
  get `bm-abc123` / `nl-abc123`. The consolidation is fine but the prefix
  is now decorative-or-debug-only depending on environment. If the intent
  is debuggable IDs, prepend `prefix-` to the UUID branch too; if not,
  drop the parameter. (Pre-existing inconsistency, just made more visible
  by the dedup.)

- **`apps/web/src/shell/drawers/EventsDrawer.tsx:148` — no test added for
  the editing/non-editing branch split.** `EventsDrawer.test.tsx` was not
  touched. The change correctly removes the invalid nested-button HTML, but
  there's no unit assertion that (a) the row reverts to a `<button>` after
  blur or (b) the rename input is still focusable when the row is in
  out-of-range state. Worth a regression test — the structural split is
  exactly the kind of thing that drifts six months later.

- **`apps/web/src/panels/VideoPanel.tsx:363` — `SETCURSOR_DELTA_NS` is
  declared inside the `useEffect`.** It's a `const bigint` with no
  dependency on closure state, so it could live at module scope alongside
  `LOOKAHEAD_NS`. Trivial.
