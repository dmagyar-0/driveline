### Medium

- **`apps/web/src/shell/Shell.module.css:4` and `apps/web/src/shell/Shell.module.css:42` — top-bar / transport heights duplicated as raw px.**
  `grid-template-rows: 36px 1fr 32px` (line 4) and `.dragOverlay { inset: 36px 0 32px 0 }` (line 42) both hard-code the same two pixel values. If anyone tweaks the top-bar height, the drag overlay silently misaligns by exactly that delta — there's no test that pins it. **Fix:** promote `--shell-topbar-h: 36px` and `--shell-transport-h: 32px` to `apps/web/src/styles/tokens.css` and consume them in both rules. Same idea would let the `drawer.module.css:31` `top: 0` / `bottom: 0` override stay correct if the rows ever resize.

- **`apps/web/src/styles/tokens.css:65` and `apps/web/src/styles/tokens.css:99` — `--shadow-popover` and `--shadow-1` are byte-identical aliases.**
  Both expand to `0 8px 24px rgba(0, 0, 0, 0.5)`. `Drawer.module.css:30` reads `--shadow-1`, `ChannelPicker.module.css:4` reads `--shadow-popover`. The duplicate guarantees they'll drift the first time someone tweaks one. **Fix:** keep `--shadow-popover` (the semantic name) and re-define `--shadow-1: var(--shadow-popover)` (or vice versa, picking whichever existing call-site wins). Don't carry both as independent literals.

### Low

- **`apps/web/src/state/persist/ui.ts:108-119` — store subscriber fires on every state change (cursor ticks at ~60 Hz).**
  `store.subscribe(...)` without a selector wakes the callback for every `set(...)` — including the `setCursorNs` ticks during playback. The two strict-equality checks short-circuit cleanly so the cost is small, but it's a wakeup-per-frame for a cold path. The same pattern is in `apps/web/src/layout/persist.ts:122-132`, so this is consistent — flagging both for follow-up. **Fix (joint):** subscribe with a selector that returns the snapshot key (`zustand/middleware`'s `subscribeWithSelector`), or move the persist hop behind a `requestIdleCallback` debouncer.

- **`apps/web/src/panels/ChannelPicker.module.css:99` — `color: #a0a0a0` missed by the Phase 0 token sweep.**
  The surrounding rules in this file all moved to `var(--color-fg-*)`. The closest token is `--color-fg-4` (`#aaa`) — pick that or add a new `--color-fg-3.5` if the half-step matters. Same review for sibling sites:
  - `apps/web/src/panels/VideoPanel.module.css:22` — `color: #ddd` (closest: `--color-fg-3` `#bbb` or `--color-fg-2` `#e0e0e0`).
  - `apps/web/src/panels/VideoPanelContainer.module.css:19` — `border: 1px solid #444` (closest: `--color-border-strong` `#333` or `--color-border-hover` `#555`).
  None of these are functional bugs; they break the "single source of truth" invariant the token file was added to enforce.

- **`apps/web/src/shell/Drawer.module.css:27-33` — narrow-viewport drawer assumes the rail is always present.**
  At `<700px` the drawer is `position: absolute; left: 40px` (cleared by the rail width). But `Rail.tsx:74` returns `null` when `railCollapsed` is `true`, leaving a 40 px dead band to the left of the drawer at narrow widths with the rail collapsed. **Fix:** thread `railCollapsed` into the drawer's class or have the rail render an invisible 40 px spacer when collapsed (current behaviour: `display: none` removes it from the flexbox).

- **`apps/web/src/timeline/Transport.module.css:122` — semantic mis-token.**
  `color: var(--color-border-hover)` substitutes a *border* token for a *text* color. The hex (`#555`) matches the original literal, but the next person who tweaks `--color-border-hover` will accidentally re-style this label. **Fix:** introduce or use a `--color-fg-*` token (e.g. add `--color-fg-7: #555` if there's no existing match).

- **`apps/web/src/timeline/Transport.module.css:60` — `.trackStrip { background: var(--color-fg-2) }` reuses a foreground token as a background.**
  Same hex (`#e0e0e0`); same semantic issue as above. The transport strip is a UI surface, not text. **Fix:** add a dedicated `--color-track-strip` token in `tokens.css`.

### Nits

- **`apps/web/src/shell/TopBar.tsx:25` — multi-key derivation in a render-on-every-tick component.**
  `cursorNs` updates per rAF, so `TopBar` re-renders ~60 ×/s during playback. The derivations (`formatRelative(...)`, the `sourceLabel` template) are cheap — just calling out that the frontend skill rule "single-key selectors only" is followed but the *re-render frequency* is high here. Memoizing `elapsed` with a `useMemo([cursorNs, startNs])` doesn't help (one input changes per tick); leaving as-is is fine.

- **`apps/web/src/shell/Drawer.tsx:18` — redundant `toLowerCase()`.**
  `STUBS` keys are already lower-case and typed as `RailTab`, so `${title.toLowerCase()}` is defensive against a value that can't exist. Use the `RailTab` key directly: `data-testid={`drawer-${activeRailTab}`}`.

- **`apps/web/src/shell/Rail.tsx:48-86` — five inline SVGs (~40 lines).**
  Acceptable for Phase 1; if Phase 2-5 land more icons, hoist to `shell/icons.tsx`.

- **`apps/web/src/state/persist/ui.ts:33-39` — `RAIL_TABS` is a runtime mirror of the `RailTab` union.**
  Out-of-band drift risk. Common idiom: derive the type from the array — `export type RailTab = (typeof RAIL_TABS)[number];` — so the type and the runtime list can never disagree.

- **`apps/web/src/App.module.css:5-17` — the `legacyShim` rule uses the deprecated `clip: rect(0 0 0 0)` form.**
  Modern equivalent is `clip-path: inset(50%)`. Since the comment promises the whole file dies in Phase 2, the lint isn't worth a flush — just keep on the radar so the SR-only idiom doesn't get copy-pasted into a new file.

- **`.claude/settings.json:8-44` — allowlist tightening drops some recurring git verbs.**
  Healthy direction (replaced `Bash(git *)` with subcommand-scoped entries). `git rm`, `git tag`, `git merge`, and `git rebase` are absent — likely intentional, but the `chore(review)` workflow uses `git push` and `git commit` only, so this should hold. Worth a re-pass before the next workflow that needs one of those.

- **`docs/design/wireframe-bundle/project/Driveline Wireframes.html` — file name contains a space.**
  Tools that don't quote (and `find -name '* *'`-style scripts) can stumble. The doc bundle is offline reference material, so low impact.
